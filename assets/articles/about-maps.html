About Maps

Each map is a JSON file and an image. Don't worry if you don't know what JSON is, you can just straight copy a map JSON from here, open it up with notepad and change the data fields to match what you're making. If you open any of these map files you will see a bunch of fields of data to be edited. This article will briefly explain them.

About name

The name field is the map's title. This is what will show up on the maps index page. The filename is also important and is similar to articles. The filename is what will be used to link to your map, so if you have a map file called island-of-moon-shards.json, you will link to it with an anchor with a tomap="island-of-moon-shards" attribute to it. More about links on the articles page.

About image

The image field is the image file for this map. All images must be inside the assets/images folder.

About links

This is a list of every link on your map. If you see any map on this, you will see that it's the image with a bunch of links on top. The idea is that these links will act like the name of the places they're on, and they're linking to an article explaining that place. Every link has a name, a size, a pos and they can have toarticle and/or tomap fields.

Again, read about links on the articles page, but you can also learn everything about links by seeing the source file of any article, including this one on assets/articles/about-maps.html.

Link names

This name is what will be inside the a tag, so it's what's going to show up on the map's page and be read by humans.

Link sizes

There are three sizes: large, medium and small. They're just for style. You can see how all of them look like by accessing a map and checking its source file.

Link pos

As you might have guessed, it stands for position. It is the position of the link in relation to the image in percentages of its size. It starts from the top left and goes to the bottom right, so a position of "x": "50%", "y": "50%" means the exact center of the map, while something like "x": "20%", "y": "80%" is close to the bottom left.

All positions must have both the x and y fields, and all links must have a position.

Link rotation

This is an optional field, you can just not include it if you don't want to. If you use it, put a number on it and it will rotate your link by that number in degrees, so 90 will make it vertical, 45 is perfectly diagonal and so on. You can put negative numbers in there too, so numbers from 90 to -90 should cover all angles.

Link radius

This is an optional field, you can just not include it if you don't want to. If you use it, put a number on it and it will curve the link text as if it was written on the diameter of a circle with radius equal to the number you put in, in pixels. That means the bigger the number, the less curved the text will be. A negative radius curves the text the other way.

Link tomaps and toarticles

These are the filenames of the article/map you're trying to link to. If you've seen the source file of any article and seen how links that link to an article or map inside the page look like, it's the same thing.

A link with "toarticle": "about-maps" should link to this article right here, while a link with "tomap": "map2" should link to Example Map Two. You don't need to have one or the other, you can have neither or even both at the same time, but having links without either should look weird as your players might think they are broken, so use them with care.

About JSON

Lastly, I wanted to talk about JSON. If you have never seen this file type, don't worry. I'm just writing this section here because they are very easy to mess up and might be a source of frustration with your maps not loading. Every field name needs to be inside quotation marks, and if you forget to close a quotation mark, your map won't load. Same thing for the value of each field, at least for these map JSONs, these values must be inside quotation marks.

Also, after every pair of field/value, there must be a comma except after the last one. You can see that after the name and the image, there is a comma. Forgetting that comma will cause your map to not load. You can also see a comma after every curly braces at the end of every link entry as well, and as you might have guessed it, forgetting these will cause your map to break.

Except for the last entries. You can see that after the last link, there is no comma at its closing curly braces. Adding a comma here will cause your map to not load. Yes, I know it's annoying but it's the easiest way to interface with browsers and the internet right now. Maybe in the future if it becomes enough of a pain point I might add support for a better data storage means, but right now I think it's good enough.

For all these reasons, I recommend you only make maps by copying working maps and editing out it's values. You can add more links by copying an entire block and adding them to the list, just be careful when adding to the bottom of the list. You can just add them to the top of the list instead, the order of the links in this link won't matter since you manually set their positions anyway. Just come back to this article if any map breaks and carefully comb your file.
