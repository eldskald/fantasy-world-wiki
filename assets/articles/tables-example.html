About table classes

You might have noticed that the above tables in the source file have a bunch of classes besides left-float and right-float. Do you need to put every single one of them on all your tables? What do they do? They're here to help you customize the columns.

This one has a left-col-2 class on it. This means align to the left the text on column 2. You can deduce that left-col-1 means alin left the first column, and you would be right. All columns are by default with center alignment. You can put the text-left class on the table to align all columns to left, and then you can add a center-col-2 to center only the second column.

About table HTML syntax

You might have notice that the syntax for tables is pretty big and verbose. Don't let it scare you, it's pretty simple. The tbody tag has the whole table. The tr tag stands for row, and you can have as many of these as you want, but inside each one of these you should keep the same number of td tags, they stand for data but in here, they kinda mean column. So for a table with 4 rows and 3 columns, you'll want 4 tr tags each with 3 td tags.

Actually, you'll need the first row to have a title of each column, right? Just add another tr tag on top with th instead of td tags. They stand for header and they will make whatever you put inside them to look like the headers in these examples. So in the case of the 4 by 3 table above, you will end up having 5 tr tags instead of 4, with the first one containing 3 th tags with the names of the columns.

There are also these thead and caption tags in the examples above. You can deduce they're for the title of the entire table and the caption explanation. You can just straight copy them to include a title and/or caption to your table. The colspan value should be the total number of columns in your table.
